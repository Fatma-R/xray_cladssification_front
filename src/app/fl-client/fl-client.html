How to Get Started

GPU Version — Using Docker with NVIDIA GPU Support 
Prepare your dataset folder
  Place your dataset locally, e.g.:
  C:\Users\safio\Desktop\clients\client_0

Ensure prerequisites:
- Docker Desktop installed and running
- NVIDIA GPU drivers installed
- NVIDIA Container Toolkit installed

Verify GPU access inside Docker:
  docker run --rm --gpus all nvidia/cuda:12.1.1-base-ubuntu22.04 nvidia-smi

Build your Docker image:
  docker build -t flower-client-gpu .

Run with GPU support and mounted volume:
  docker run --rm --gpus all -v "C:/Users/safio/Desktop/clients/client_0:/app/dataset" flower-client-gpu --dataset_path /app/dataset

Logs and Output:
  Client loads dataset, uses GPU, connects to federated server
CPU Version — Using Docker Without GPU 
Prepare your dataset folder
  Example path:
  C:\Users\safio\Desktop\clients\client_0

Ensure Docker is installed and running

Build CPU-only image:
  docker build -t flower-client-cpu .

Run the container:
  docker run --rm -v "C:/Users/safio/Desktop/clients/client_0:/app/dataset" flower-client-cpu --dataset_path /app/dataset

Monitor output and logs.
Notes
  • GPU version needs NVIDIA drivers and container toolkit
  • CPU version is slower but works everywhere Docker runs
  • Use --dataset_path to specify dataset folder
